Tips

  • কোড পিসিতে লিখে Practice করা। অনেকেই কোড খাতায় লেখে মুখস্থ করার চেষ্টা করে। এভাবে কোডের ভুলগুলো সহজে ধরা পড়ে না। পিসিতে লিখে compile করলে ভুলগুলো ধরা পড়ে। ভুল থেকে বেশি শেখা যায়।
  • অনেক সময় Bracket/Parenthesis highlight করে না, Bracket close হয়ছে কিনা indicate করে না।। এজন্য Edit>Preferences>View এ যেয়ে Highlight matching brackets/parenthesis এ ✔ দিয়ে নিতে হবে। [Ubuntu তে Gedit এর জন্য প্রযোজ্য ]
  • অনেক বড় function call করার সময় function ছোট ছোট অংশে ভেঙে আলাদা আলাদা variable দিয়ে প্রকাশ করলে ভুল হওয়ার সম্ভাবনা অনেকটাই কমে আসে।
  • কোডে ঠিকঠাক spacing ব্যবহার করা। ঠিকঠাক spacing ব্যবহার করলে ভুলগুলো সহজে চোখে পড়ে।
  • প্রয়োজনীয় সূত্রগুলো আলাদাভাবে লিখে রাখা। কোডে সূত্রগুলো ব্র‍্যাকেট ব্যবহার করে এক লাইনেই লেখা হয়, যেটা মনে রাখা কঠিন।
  • আরো একটা ভাল উপায় হচ্ছে, কোনো solved code দেখে দেখে লেখা। অনেক কষ্ট করে কোনো প্রবলেমের কোড করতে যত সময় লাগে, ঐ প্রবলেমের solved code দেখে দেখে দ্রুত শেখা যায়। Resources সেকশনে ল্যাবের দেয়া pdf sheet গুলার code এবং এগুলোর data plot করার জন্য plt file সব দেয়া আছে।
  • Integration এর সময় যে variable এর respect এ integrate করতে হয় function এর argument এ ঐ variable বাদে বাকি variable দের call করতে হয়।
  • কোডের কোন অংশের কাজ কি সেটা comment আকারে লিখে রাখা বুদ্ধিমানের কাজ। এতে code এর readability বাড়ে।
  • একদমই Basic পর্যায়ে সমস্যা থাকলে SoloLearn App টা অনেক ভাল। Available at PlayStore.
