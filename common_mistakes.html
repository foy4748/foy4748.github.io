Common Mistakes

  • Function and Array


    1. ব্র‍্যাকেট/Parenthesis ঠিকমত close না করা, ভুল প্রয়োগ করা।

    2. অ্যারের Dimension এর চেয়ে বেশি value input দিলে output ভুল আসে, এরর মেসেজ আসে না।

    3. return type function এ input হিসেবে Array elements গুলা ব্যবহার করা গেলেও output এ Array return করতে পারে না। এক্ষেত্রে void type function কাজে আসে।

    4. অনেক সময় function এর value infinity হওয়ায় data plot করার পর graph ঠিকঠাক আসে না। Example: tanX, X[-π, π] এ ক্ষেত্রে y axis এ output limit set করে দিতে হয়। এর জন্য gnuplot এ data plot করার আগে set yrange [-5:5] লিখে enter চাপতে হয়, যাতে y axis এ -5 থেকে 5 এর মধ্যে থাকা function এর মান plot হয়।

    5. Loop এ calculate করে ফলাফল প্রতি iteration এ diverge করে দেয় এমন কাজ করা যাবে না। যেমন
      { sum += 2*f(a + i*h): } ✖
      {sum += f(a+ i*h); } result = 2*sum; ✔✔

    6. Loop এ call করা variable loop এর body তে কোথাও call না করলে loop রান করে না, যার কারণে ফলাফল ভুল আসে। তারচেয়েও চিন্তার বিষয়, terminal এ কোনো error message আসে না। Example:
      for(i=0;i<n; i++) { result += 1/pow(n,2) } ✖
      for(i=0;i<n;i++) { result += 1/pow(i,2) } ✔



  • Root finding


    1. Newton-Raphson method এ derivative ঠিকমত নির্ণয় না করা। Example:
      ⇒ X = tanX
      ⇒ X = sinX/cosX
      Case #1 Case #2
      ⇒ XcosX - sinX = 0 -----(1)
      ⇒ -XsinX + cosX - cosX = 0
      ⇒ -XsinX = 0       		⇒ sinX - XcosX = 0 -----(2)
      ⇒ cosX + XsinX - cosX = 0
      ⇒ XsinX = 0
      যদি function টা (1) এর মত লেখা হয় তাহলে function টার derivative এ উভয় পক্ষে -1 গুণ করা যাবে না, কারণ 1st derivative এ রকম রেসাল্টই আসে।যদি function (2) এর মত লেখা হয় তবে তার corresponding 1st derivative এ মাইনাস চিহ্নটা আসে না!

    2. Bisection method এ actual root এর কাছাকাছি মান interval এ রাখা,NewtonRaphson method এ approximate root actual root এর কাছাকাছি না রাখা। এই ভুল করলে ঠিকঠাক কোড লিখেও লাভ নেই।



  • Integration


    1. h = (b-a)/N আর প্রবলেমে উল্লেখ থাকা n সংখ্যক term এর Summation এ N আর n একই variable দিয়ে লেখা! h নির্ণয়ের জন্য fixed একটা N (number of intervals) থাকে, আর n সংখ্যক term কে যোগ করার জন্য n কে Loop এ রেখে বাড়াতে হয়। দুটাই N বা n দিয়ে লেখা সাধারণ ভুল।

    2. একাধিকবার Integration এর ক্ষেত্রে function এর Arguments ঠিকঠাক না লেখা।

    3. Gaussian Quadrature এ সবগুলো অপারেশন একটা লুপেই করা।

    4. যদি function এর মান Limit এর কোথাও infinity হয়ে যায় তাহলে Simpson's rule কোনো কাজেই আসে না, এরজন্য Gaussian Quadrature method apply করতে হয়। Example: (1 + cos²X)/sinX এই function 0→π লিমিটে integration করলে 0 এবং π উভয় ক্ষেত্রেই function এর মান infinity আসে, কাজেই Gaussian Quadrature method use করতে হবে।



  • Solving Differential Equations


    1. অনেক সময় Initial ভ্যালুতে Function এর মান Infitinity হয়। উদাহরণ হিসেবে Lane Emden এর DE এর কথা বলা যেতে পারে। এখানে dy/dx = -(2/x)*v - ys ; y(0) = 1, v(0) = 0; অর্থাৎ, x = 0 এর জন্য Initial value গুলো দেয়া, কিন্তু function এর মান x= 0 তে infinity আসে, কাজেই x এর মান শূন্যর কাছাকাছি ধরতে হবে।

    2. দুইটি Coupled 2nd Order DE solve করার ক্ষেত্রে চারটি 1st Order DE solve করতে হয়, যেখানে দুই ধরনের Velocity -- (1) Linear ও (2) Angular velocity update করার প্রয়োজন হয়। যেহেতু DE গুলো coupled কাজেই algorithm গুলো দেখতে অনেকটা নিম্নরূপঃ

      v[i+1] = v[i] + h*G(L[i], theta[i], omega[i]) -------------------------(i)
      omega[i+1] = omega[i] + h*F(L[i][, v[i], theta[i], omega[i] -----(ii)

      ২য় লাইনে স্পষ্টত Angular velocity update করতে Linear Velocity পুরোনোটাই ব্যবহার হচ্ছে। কিন্তু আমরা যদি ১ম লাইনেই Linear Velocity আপডেট করে ফেলি তাহলে Angular Velocity update করতে অনিচ্ছাকৃতভাবে Updated Linear Velocity induce হবে। এ ক্ষেত্রে উপরের সমীকরণে ব্যবহৃত Array Indexing ব্যবহার করলে এরকম সমস্যা হয় না। তবে array indexing ব্যবহার না করলে Loop এ দ্বিতীয় সমীকরণটা আগে calculate করতে হবে।
